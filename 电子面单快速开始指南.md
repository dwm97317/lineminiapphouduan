## 🚀 快速开始

### 1. 初始化数据库

在项目根目录执行：

```bash
php init_waybill_database.php
```

这将自动创建：
- `yoshop_waybill_record` 表（面单打印记录）
- 中通和顺丰的默认配置数据

### 2. 访问配置页面

在浏览器中打开：

```
http://你的域名/index.php?s=/store/setting.waybill_config/index
```

### 3. 配置步骤

#### API 配置（第一个选项卡）

**中通快递：**
1. 填写 API 地址：`https://api.zhongtong.com`
2. 填写 API Key 和 API Secret
3. 点击"保存 API 配置"

**顺丰快递：**
1. 填写 API 地址：
   - 生产：`https://bspgw.sf-express.com/std/service`
   - 沙箱：`https://sfapi-sbox.sf-express.com/std/service`
2. 填写合作伙伴编码（Partner ID）
3. 填写校验码/密钥（Checkword）
4. 填写月结卡号（可选）
5. 选择付款方式
6. 点击"保存 API 配置"

#### 面单配置（第二个选项卡）

1. 选择快递公司（中通/顺丰）
2. 勾选需要显示的字段
3. 填写快递公司特定字段：
   - **中通**：网点代码、网点名称
   - **顺丰**：月结卡号、付款方式
4. 设置打印参数：
   - 纸张大小：76mm x 130mm
   - 打印方向：纵向/横向
   - 缩放比例：50% - 150%
5. 点击"保存配置"

## 💾 数据库结构

### yoshop_waybill_record（面单记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| inpack_id | int | 集运订单ID |
| order_sn | varchar(50) | 订单号 |
| express_type | varchar(20) | 快递类型 |
| express_name | varchar(50) | 快递公司名称 |
| waybill_no | varchar(50) | 运单号 |
| operation_type | tinyint | 操作类型（1:打印 2:下单） |
| print_time | int | 打印时间 |
| created_time | int | 创建时间 |

### yoshop_setting（配置表）

配置项 `key` 值：
- `waybill` - API 配置
- `waybill_config_zhongtong` - 中通面单配置
- `waybill_config_shunfeng` - 顺丰面单配置

## 📝 API 接口

### 获取配置

```http
GET /store/setting.waybill_config/getConfig?express_type=zhongtong
```

### 保存配置

```http
POST /store/setting.waybill_config/saveConfig
Content-Type: application/x-www-form-urlencoded

express_type=zhongtong&config={"fields":{...}}
```

### 获取字段列表

```http
GET /store/setting.waybill_config/getFieldList?express_type=zhongtong
```

### 恢复默认

```http
POST /store/setting.waybill_config/resetConfig
Content-Type: application/x-www-form-urlencoded

express_type=zhongtong
```

## ⚠️ 注意事项

1. **wxapp_id**：默认使用 10001，根据实际情况调整
2. **缓存**：配置保存后会自动清除缓存
3. **权限**：确保数据库用户有创建表和写入权限
4. **编码**：所有数据使用 UTF-8 编码

## 🐛 常见问题

### Q: 保存后配置没生效？
**A:** 手动清空 `runtime/cache` 目录

### Q: 数据库连接失败？
**A:** 检查 `source/application/database.php` 配置

### Q: 静态方法冲突错误？
**A:** 已修复。WaybillConfigService 使用实例方法保存配置
