# Feature Specification: 快递面单打印功能

**Feature Branch**: `004-express-waybill-printing`
**Created**: 2026-01-17
**Status**: Draft

## 用户场景与测试

### User Story 1 - 查看打印按钮 (Priority: P0)

作为仓库管理员，我需要在订单列表的操作区域看到"打印中通"和"打印顺丰"两个按钮，以便快速选择快递公司打印面单。

**Why this priority**: 这是核心功能的入口，没有按钮用户无法使用打印功能。

**Independent Test**: 
1. 登录后台管理系统
2. 访问 http://localhost:8080/index.php?s=/store/tr_order/all_list
3. 在订单列表的操作列中应该能看到两个新按钮

**Acceptance Scenarios**:
1. **Given** 用户已登录后台系统，**When** 访问全部订单列表页面，**Then** 每个订单的操作区域显示"打印中通"和"打印顺丰"按钮
2. **Given** 订单列表有多条记录，**When** 滚动查看不同订单，**Then** 所有订单都显示这两个按钮
3. **Given** 按钮已显示，**When** 鼠标悬停在按钮上，**Then** 按钮有视觉反馈（hover 效果）

---

### User Story 2 - 预览中通面单 (Priority: P0)

作为仓库管理员，当我点击"打印中通"按钮后，系统应该调用中通快递 API 获取面单数据，并在弹窗中显示面单预览，让我确认信息无误后再决定是否打印。

**Why this priority**: 预览功能是核心流程，确保打印前信息正确，避免浪费面单。

**Independent Test**:
1. 在订单列表点击某个订单的"打印中通"按钮
2. 系统应弹出预览窗口显示中通面单
3. 面单应包含完整的寄件人、收件人、条形码等信息

**Acceptance Scenarios**:
1. **Given** 用户点击"打印中通"按钮，**When** API 调用成功，**Then** 弹出预览窗口显示中通面单样式
2. **Given** 预览窗口已打开，**When** 查看面单内容，**Then** 显示订单号、寄件人信息、收件人信息、条形码、重量等完整信息
3. **Given** API 调用失败，**When** 网络错误或数据异常，**Then** 显示友好的错误提示信息
4. **Given** 预览窗口已打开，**When** 点击窗口外部或关闭按钮，**Then** 预览窗口关闭

---

### User Story 3 - 预览顺丰面单 (Priority: P0)

作为仓库管理员，当我点击"打印顺丰"按钮后，系统应该调用顺丰快递 API 获取面单数据，并在弹窗中显示面单预览，让我确认信息无误后再决定是否打印。

**Why this priority**: 与中通功能同等重要，支持多快递公司是业务需求。

**Independent Test**:
1. 在订单列表点击某个订单的"打印顺丰"按钮
2. 系统应弹出预览窗口显示顺丰面单
3. 面单应包含完整的寄件人、收件人、条形码等信息

**Acceptance Scenarios**:
1. **Given** 用户点击"打印顺丰"按钮，**When** API 调用成功，**Then** 弹出预览窗口显示顺丰面单样式
2. **Given** 预览窗口已打开，**When** 查看面单内容，**Then** 显示订单号、寄件人信息、收件人信息、条形码、重量等完整信息
3. **Given** API 调用失败，**When** 网络错误或数据异常，**Then** 显示友好的错误提示信息
4. **Given** 预览窗口已打开，**When** 点击窗口外部或关闭按钮，**Then** 预览窗口关闭

---

### User Story 4 - 立即打印操作 (Priority: P1)

作为仓库管理员，在预览面单后，我需要点击"立即打印"按钮来打印面单，系统应该调用浏览器打印功能或直接发送到打印机。

**Why this priority**: 这是完成打印流程的关键步骤。

**Independent Test**:
1. 打开面单预览窗口
2. 点击"立即打印"按钮
3. 浏览器应弹出打印对话框或直接打印

**Acceptance Scenarios**:
1. **Given** 预览窗口已打开，**When** 点击"立即打印"按钮，**Then** 触发浏览器打印功能
2. **Given** 打印对话框已打开，**When** 用户确认打印，**Then** 面单发送到打印机
3. **Given** 打印成功，**When** 打印完成，**Then** 系统记录打印日志（订单ID、快递公司、打印时间）
4. **Given** 打印成功，**When** 打印完成，**Then** 预览窗口自动关闭或显示成功提示

---

### User Story 5 - 只下单不打印 (Priority: P1)

作为仓库管理员，有时我只需要在快递公司系统中创建订单获取运单号，但暂时不打印面单（比如批量处理后统一打印），所以需要"只下单"功能。

**Why this priority**: 提供灵活的工作流程，支持批量操作场景。

**Independent Test**:
1. 打开面单预览窗口
2. 点击"只下单"按钮
3. 系统应调用快递 API 创建订单但不触发打印

**Acceptance Scenarios**:
1. **Given** 预览窗口已打开，**When** 点击"只下单"按钮，**Then** 调用快递 API 创建订单
2. **Given** 下单成功，**When** API 返回运单号，**Then** 系统保存运单号到订单记录
3. **Given** 下单成功，**When** 操作完成，**Then** 显示成功提示"已下单，运单号：XXXXXX"
4. **Given** 下单成功，**When** 操作完成，**Then** 预览窗口关闭
5. **Given** 下单失败，**When** API 返回错误，**Then** 显示错误信息但不关闭预览窗口

---

### User Story 6 - 取消操作 (Priority: P2)

作为仓库管理员，如果我在预览面单后发现信息有误或暂时不需要打印，我需要能够取消操作并关闭预览窗口。

**Why this priority**: 基本的用户体验需求，允许用户退出操作。

**Independent Test**:
1. 打开面单预览窗口
2. 点击"取消"按钮
3. 预览窗口应关闭且不执行任何操作

**Acceptance Scenarios**:
1. **Given** 预览窗口已打开，**When** 点击"取消"按钮，**Then** 预览窗口关闭
2. **Given** 预览窗口已打开，**When** 点击"取消"按钮，**Then** 不调用任何 API
3. **Given** 预览窗口已打开，**When** 点击"取消"按钮，**Then** 订单状态不发生变化
4. **Given** 预览窗口已打开，**When** 按 ESC 键，**Then** 预览窗口关闭（等同于取消）

---

### Edge Cases

- **订单信息不完整**: 如果订单缺少 address_id 或地址信息不完整（姓名、电话、地址为空），点击打印按钮时应该提示"订单收货信息不完整，请先完善收货地址"
- **地址不存在**: 如果订单的 address_id 在 user_address 表中不存在，应提示"收货地址不存在，请重新选择地址"
- **重复打印**: 如果订单已经打印过面单，再次点击打印按钮时应该提示"该订单已打印过面单（运单号：XXX），是否重新打印？"
- **快递 API 超时**: 如果快递 API 响应超过 30 秒，应该显示超时提示并允许用户重试
- **打印机未连接**: 如果用户点击"立即打印"但系统检测不到打印机，应该提示"未检测到打印机，请检查打印机连接"
- **并发打印**: 如果用户同时打开多个预览窗口，每个窗口应该独立操作，互不影响
- **权限控制**: 只有具有"打印面单"权限的用户才能看到打印按钮
- **配置未初始化**: 如果快递公司配置不存在，系统应自动使用默认配置
- **配置冲突**: 如果配置的字段与快递公司 API 要求冲突，应提示管理员调整配置
- **字段依赖**: 某些字段可能相互依赖（如显示重量必须显示单位），系统应自动处理这些依赖关系

## Requirements

### Functional Requirements

#### 打印功能
- **FR-001**: 系统必须在 `/store/tr_order/all_list` 页面的每个订单操作区域显示"打印中通"和"打印顺丰"两个按钮
- **FR-002**: 系统必须在订单列表顶部提供批量操作区域，包含"批量打印中通"和"批量打印顺丰"按钮
- **FR-003**: 点击"打印中通"按钮时，系统必须调用中通快递 API 获取面单数据
- **FR-004**: 点击"打印顺丰"按钮时，系统必须调用顺丰快递 API 获取面单数据
- **FR-005**: 系统必须在弹窗中显示面单预览，包含寄件人、收件人、条形码、重量等完整信息
- **FR-006**: 预览窗口必须提供三个操作按钮："立即打印"、"只下单"、"取消"
- **FR-007**: 批量打印时，预览窗口必须提供四个操作按钮："打印并继续"、"只下单并继续"、"跳过"、"取消批量"
- **FR-008**: 点击"立即打印"按钮时，系统必须触发浏览器打印功能
- **FR-009**: 点击"只下单"按钮时，系统必须调用快递 API 创建订单并保存运单号
- **FR-010**: 点击"取消"按钮时，系统必须关闭预览窗口且不执行任何操作
- **FR-011**: 批量打印完成后，系统必须显示统计信息（成功数、跳过数、失败数）
- **FR-012**: 系统必须记录每次打印操作的日志（订单ID、快递公司、运单号、操作时间、操作人）
- **FR-013**: 系统必须在订单详情页显示该订单的打印历史记录
- **FR-014**: 系统必须在打印前验证订单信息完整性（address_id 不能为空，收件人姓名、电话、地址不能为空）
- **FR-015**: 系统必须支持重复打印，并在重复打印时给出提示
- **FR-016**: 系统必须处理 API 调用失败的情况，显示友好的错误提示

#### 配置管理功能
- **FR-017**: 系统必须提供面单配置管理页面 `/store/setting/waybill_config`
- **FR-018**: 配置页面必须支持选择快递公司（中通、顺丰）
- **FR-019**: 系统必须支持配置每个快递公司的字段显示/隐藏（如备注、重量、体积、物品名称等）
- **FR-020**: 系统必须支持配置快递公司特定字段（中通：网点代码、网点名称；顺丰：月结卡号、付款方式）
- **FR-021**: 系统必须支持配置打印参数（纸张大小、打印方向、缩放比例）
- **FR-022**: 配置页面必须提供"保存"和"恢复默认"按钮
- **FR-023**: 系统必须提供 API 接口获取指定快递公司的配置信息
- **FR-024**: 系统必须提供 API 接口保存快递公司配置信息
- **FR-025**: 系统必须提供 API 接口获取快递公司支持的字段列表
- **FR-026**: 系统必须提供 API 接口恢复快递公司配置为默认值
- **FR-027**: 面单生成时必须根据配置动态显示/隐藏字段
- **FR-028**: 打印时必须应用配置的打印参数（纸张大小、方向、缩放）
- **FR-029**: 配置数据必须存储在 Setting 表中，使用 JSON 格式
- **FR-030**: 系统必须为每个快递公司提供合理的默认配置

### Key Entities

- **订单 (Inpack)**: 集运订单，包含订单号、用户ID (member_id)、地址ID (address_id)、重量、运费等
- **用户地址 (UserAddress)**: 用户收货地址，通过 address_id 关联，包含姓名、电话、省市区、详细地址、邮编等
- **快递公司 (Express)**: 快递公司信息，包含公司名称、API 配置、面单模板等
- **运单记录 (WaybillRecord)**: 打印记录，包含订单ID、快递公司、运单号、打印时间、操作人等
- **面单配置 (WaybillConfig)**: 快递公司面单配置，包含字段显示设置、打印参数、快递公司特定字段等

#### 数据关联关系

```
Inpack (集运订单)
  ├─ member_id → User (用户)
  └─ address_id → UserAddress (收货地址)
      ├─ name (收件人姓名)
      ├─ phone (收件人电话)
      ├─ country (国家)
      ├─ province (省份)
      ├─ city (城市)
      ├─ region (区域)
      ├─ detail (详细地址)
      ├─ street (街道)
      ├─ door (门牌号)
      ├─ code (邮编)
      ├─ email (邮箱)
      ├─ identitycard (身份证号)
      └─ clearancecode (清关码)
```

**面单收货信息来源**：
- 面单上的收货信息来自该集运订单 (Inpack) 的 address_id 字段
- 通过 address_id 查询 user_address 表获取完整的收货地址信息
- 包含收件人姓名、电话、完整地址（国家、省、市、区、详细地址、街道、门牌号）、邮编、邮箱等

## Success Criteria

### Measurable Outcomes

#### 打印功能
- **SC-001**: 用户能在 3 秒内在订单列表页面看到打印按钮
- **SC-002**: 点击打印按钮后，面单预览窗口在 5 秒内显示（正常网络环境）
- **SC-003**: 面单预览窗口显示的信息准确率达到 100%（与订单数据一致）
- **SC-004**: 用户能在预览后 2 秒内完成打印或下单操作
- **SC-005**: 系统能正确处理 95% 以上的快递 API 调用（成功率）
- **SC-006**: 所有打印操作都有日志记录，日志完整率 100%
- **SC-007**: 用户能在 1 秒内取消预览操作并关闭窗口

#### 配置管理功能
- **SC-008**: 管理员能在 3 秒内访问配置页面并看到当前配置
- **SC-009**: 配置保存操作在 2 秒内完成并返回成功提示
- **SC-010**: 配置修改后，下次打印立即生效，无需重启系统
- **SC-011**: 配置 API 响应时间 < 1 秒
- **SC-012**: 配置数据验证准确率 100%（拒绝无效配置）
- **SC-013**: 恢复默认配置操作在 1 秒内完成
- **SC-014**: 面单根据配置正确显示/隐藏字段，准确率 100%

### User Story 7 - 批量打印功能 (Priority: P1)

作为仓库管理员，我需要能够一次选择多个订单进行批量打印，以提高工作效率，避免逐个点击打印。

**Why this priority**: 批量操作是提升效率的关键功能，特别是在订单量大的情况下。

**Independent Test**:
1. 在订单列表勾选多个订单（如 5 个订单）
2. 点击批量操作区域的"批量打印中通"或"批量打印顺丰"按钮
3. 系统应依次显示每个订单的面单预览

**Acceptance Scenarios**:
1. **Given** 用户勾选了多个订单，**When** 点击"批量打印中通"按钮，**Then** 系统依次调用中通 API 获取每个订单的面单数据
2. **Given** 批量打印已开始，**When** 显示第一个订单的预览，**Then** 用户可以选择"打印并继续"、"跳过"或"取消批量"
3. **Given** 用户选择"打印并继续"，**When** 当前订单打印完成，**Then** 自动显示下一个订单的预览
4. **Given** 用户选择"跳过"，**When** 跳过当前订单，**Then** 直接显示下一个订单的预览
5. **Given** 用户选择"取消批量"，**When** 取消操作，**Then** 停止批量打印并关闭预览窗口
6. **Given** 所有订单处理完成，**When** 批量打印结束，**Then** 显示统计信息（成功 X 个，跳过 Y 个，失败 Z 个）

---

### User Story 8 - 面单视图配置管理 (Priority: P1)

作为系统管理员，我需要在后台配置不同快递公司的面单显示字段和打印参数，以便根据实际业务需求灵活调整面单内容和打印效果。

**Why this priority**: 不同快递公司支持的字段不同，业务需求也会变化，需要灵活配置而不是硬编码。

**Independent Test**:
1. 访问后台设置页面 `/store/setting/waybill_config`
2. 选择快递公司（中通或顺丰）
3. 配置字段显示/隐藏和打印参数
4. 保存配置后打印面单验证效果

**Acceptance Scenarios**:
1. **Given** 管理员访问面单配置页面，**When** 选择"中通快递"，**Then** 显示中通支持的所有可配置字段列表
2. **Given** 配置页面已打开，**When** 勾选/取消勾选某个字段（如"备注"），**Then** 该字段在面单预览中显示/隐藏
3. **Given** 配置页面已打开，**When** 填写快递公司特定字段（如中通网点代码），**Then** 该信息保存到配置中
4. **Given** 配置页面已打开，**When** 设置打印参数（纸张大小、打印方向、缩放比例），**Then** 打印时应用这些参数
5. **Given** 配置已修改，**When** 点击"保存"按钮，**Then** 配置保存成功并显示提示信息
6. **Given** 配置已保存，**When** 打印面单，**Then** 面单按照新配置显示字段和应用打印参数
7. **Given** 配置页面已打开，**When** 点击"恢复默认"按钮，**Then** 所有配置恢复为系统默认值

---

### User Story 9 - 面单配置 API (Priority: P1)

作为开发人员，我需要提供 RESTful API 来管理面单配置，支持获取、保存、更新配置信息。

**Why this priority**: API 是配置功能的基础，前端页面和打印功能都依赖这些 API。

**Independent Test**:
1. 使用 Postman 或 curl 调用配置 API
2. 验证返回数据格式和内容正确性
3. 测试配置保存和读取的一致性

**Acceptance Scenarios**:
1. **Given** 调用 GET `/store/setting.waybill_config/getConfig`，**When** 传入 `express_type=zhongtong`，**Then** 返回中通的完整配置 JSON
2. **Given** 调用 POST `/store/setting.waybill_config/saveConfig`，**When** 提交配置数据，**Then** 配置保存成功并返回成功状态
3. **Given** 配置数据格式错误，**When** 调用保存 API，**Then** 返回 400 错误和具体错误信息
4. **Given** 调用 GET `/store/setting.waybill_config/getFieldList`，**When** 传入 `express_type=shunfeng`，**Then** 返回顺丰支持的所有字段列表及默认值
5. **Given** 调用 POST `/store/setting.waybill_config/resetConfig`，**When** 传入快递公司类型，**Then** 该快递公司配置恢复为默认值

---

## Assumptions

- 中通和顺丰快递 API 已经配置完成，包含必要的 API 密钥和接口地址，且有测试环境可用
- 集运订单 (Inpack) 中已包含 address_id 字段，关联到用户地址表 (user_address)
- 用户地址表 (user_address) 包含完整的收货信息（姓名、电话、地址、邮编等）
- 寄件人信息从系统配置或仓库信息中获取
- 用户使用的浏览器支持 window.print() 打印功能
- 打印机已正确连接到用户电脑
- 面单模板已经设计完成，符合中通和顺丰的规范要求
- 打印日志只需在订单详情页显示，不需要独立的日志管理页面
- 批量打印时，每个订单需要逐个确认预览，不支持一键打印所有订单（避免错误）
- 所有快递公司使用统一规格的一联快递单（76mm x 130mm）
- 面单字段的显示/隐藏不影响快递公司 API 的必填字段要求
- 配置管理权限仅限系统管理员，普通用户只能使用配置好的设置打印
- 不同快递公司的配置相互独立，互不影响
