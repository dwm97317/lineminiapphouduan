# Feature Specification: 推荐奖励系统配置视图增强

**Feature Branch**: `001-referral-config-view-enhancement`
**Created**: 2026-01-17
**Status**: Draft

## User Scenarios & Testing

### User Story 1 - 任务配置可视化增强 (Priority: P1)

管理员需要更直观地查看和配置推荐任务，包括任务类型、参数、状态等信息，以便快速了解系统配置情况。

**Why this priority**: 当前配置页面信息展示较为简单，管理员难以快速了解任务的完整配置情况，增强视图可以提高配置效率。

**Independent Test**: 访问推荐系统配置页面的"任务配置"标签，查看是否显示了任务的详细信息卡片。

**Acceptance Scenarios**:
1. **Given** 管理员打开任务配置页面，**When** 查看推荐人任务列表，**Then** 每个任务显示为独立的卡片，包含任务名称、类型、状态标签、参数配置
2. **Given** 管理员查看任务卡片，**When** 任务被禁用，**Then** 卡片显示灰色状态标签"已禁用"
3. **Given** 管理员查看任务卡片，**When** 任务被设置为必须完成，**Then** 卡片显示红色标签"必须完成"
4. **Given** 管理员查看任务参数，**When** 任务有特定参数（如最低邀请人数），**Then** 参数以独立的输入框形式展示，带有单位说明

---

### User Story 2 - 奖励配置可视化增强 (Priority: P1)

管理员需要更清晰地查看和配置不同级别的推荐奖励，包括奖励类型、金额、比例等信息，以便更好地管理奖励策略。

**Why this priority**: 奖励配置是推荐系统的核心，清晰的视图可以帮助管理员避免配置错误，提高管理效率。

**Independent Test**: 访问推荐系统配置页面的"奖励配置"标签，查看是否显示了奖励的详细信息卡片。

**Acceptance Scenarios**:
1. **Given** 管理员打开奖励配置页面，**When** 查看奖励列表，**Then** 每个奖励显示为独立的卡片，包含级别标签、奖励类型图标、金额/比例信息
2. **Given** 管理员查看奖励卡片，**When** 奖励类型为现金，**Then** 显示现金图标和金额输入框，以及最低提现金额参数
3. **Given** 管理员查看奖励卡片，**When** 奖励类型为积分，**Then** 显示积分图标和数量输入框，以及积分有效期参数
4. **Given** 管理员查看奖励卡片，**When** 奖励类型为优惠券，**Then** 显示优惠券图标和优惠券ID选择器
5. **Given** 管理员查看奖励卡片，**When** 奖励被禁用，**Then** 卡片显示半透明状态，带有"已禁用"标签

---

### User Story 3 - 配置统计面板 (Priority: P2)

管理员需要在配置页面顶部看到系统配置的统计信息，包括启用的任务数量、奖励配置数量等，以便快速了解系统整体状态。

**Why this priority**: 统计面板可以提供系统配置的概览，帮助管理员快速判断系统配置是否完整。

**Independent Test**: 访问推荐系统配置页面，查看页面顶部是否显示统计面板。

**Acceptance Scenarios**:
1. **Given** 管理员打开配置页面，**When** 查看页面顶部，**Then** 显示统计面板，包含"启用任务数"、"奖励配置数"、"推荐级数"等信息
2. **Given** 管理员查看统计面板，**When** 有任务被禁用，**Then** 统计面板显示"X/Y 任务启用"的格式
3. **Given** 管理员查看统计面板，**When** 点击统计卡片，**Then** 自动跳转到对应的配置标签页

---

### User Story 4 - 配置预览模式 (Priority: P2)

管理员需要在保存配置前预览配置效果，包括任务流程、奖励计算示例等，以便验证配置的正确性。

**Why this priority**: 预览功能可以帮助管理员在保存前发现配置错误，减少配置失误。

**Independent Test**: 在任务配置或奖励配置页面，点击"预览配置"按钮，查看是否显示预览弹窗。

**Acceptance Scenarios**:
1. **Given** 管理员完成任务配置，**When** 点击"预览配置"按钮，**Then** 显示弹窗，展示任务流程图和参数说明
2. **Given** 管理员完成奖励配置，**When** 点击"预览配置"按钮，**Then** 显示弹窗，展示奖励计算示例（如：1级推荐获得X元，2级推荐获得Y元）
3. **Given** 管理员查看预览弹窗，**When** 发现配置错误，**Then** 可以关闭弹窗并修改配置

---

### User Story 5 - 配置历史记录 (Priority: P3)

管理员需要查看配置的修改历史，包括修改时间、修改人、修改内容等，以便追溯配置变更。

**Why this priority**: 配置历史可以帮助管理员了解配置的演变过程，便于问题排查和审计。

**Independent Test**: 在配置页面，点击"查看历史"按钮，查看是否显示配置历史记录。

**Acceptance Scenarios**:
1. **Given** 管理员打开配置页面，**When** 点击"查看历史"按钮，**Then** 显示配置历史列表，包含修改时间、修改人、修改内容摘要
2. **Given** 管理员查看历史列表，**When** 点击某条历史记录，**Then** 显示详细的配置对比（修改前 vs 修改后）
3. **Given** 管理员查看历史详情，**When** 需要恢复旧配置，**Then** 可以点击"恢复此配置"按钮

---

### Edge Cases

- 当任务配置为空时，如何显示？显示"暂无任务配置"提示，并提供"添加任务"按钮
- 当奖励配置为空时，如何显示？显示"暂无奖励配置"提示，并提供"添加奖励"按钮
- 当任务参数为空或格式错误时，如何处理？显示默认值，并在保存时进行验证
- 当奖励参数为空或格式错误时，如何处理？显示默认值，并在保存时进行验证
- 当配置保存失败时，如何提示？显示错误提示，并保留用户输入的数据
- 当多个管理员同时修改配置时，如何处理冲突？后保存的配置覆盖先保存的配置，并显示警告提示

## Requirements

### Functional Requirements

- **FR-001**: 系统必须为每个任务配置提供独立的卡片视图，包含任务名称、类型、状态标签、参数配置
- **FR-002**: 系统必须为每个奖励配置提供独立的卡片视图，包含级别标签、奖励类型图标、金额/比例信息
- **FR-003**: 系统必须在配置页面顶部显示统计面板，包含启用任务数、奖励配置数、推荐级数等信息
- **FR-004**: 系统必须提供配置预览功能，展示任务流程图和奖励计算示例
- **FR-005**: 系统必须提供配置历史记录功能，记录配置的修改时间、修改人、修改内容
- **FR-006**: 系统必须支持任务参数的动态配置，根据任务类型显示不同的参数输入框
- **FR-007**: 系统必须支持奖励参数的动态配置，根据奖励类型显示不同的参数输入框
- **FR-008**: 系统必须在保存配置前进行数据验证，确保配置的正确性
- **FR-009**: 系统必须在配置保存成功后显示成功提示，并刷新页面数据
- **FR-010**: 系统必须在配置保存失败后显示错误提示，并保留用户输入的数据

### Key Entities

- **任务配置 (TaskConfig)**: 推荐任务的配置信息，包含任务类型、任务名称、是否启用、是否必须完成、任务参数等
- **奖励配置 (RewardConfig)**: 推荐奖励的配置信息，包含奖励级别、奖励类型、奖励金额、奖励比例、有效期、奖励参数等
- **系统配置 (SystemConfig)**: 推荐系统的全局配置，包含最大推荐级数、推荐码长度、推荐关系失效天数等
- **配置历史 (ConfigHistory)**: 配置的修改历史记录，包含修改时间、修改人、修改内容、配置类型等

## Success Criteria

### Measurable Outcomes

- **SC-001**: 管理员可以在5秒内了解系统的整体配置状态（通过统计面板）
- **SC-002**: 管理员可以在10秒内完成单个任务配置的修改（通过卡片视图）
- **SC-003**: 管理员可以在10秒内完成单个奖励配置的修改（通过卡片视图）
- **SC-004**: 配置预览功能可以帮助管理员在保存前发现至少80%的配置错误
- **SC-005**: 配置历史记录可以追溯至少30天内的所有配置变更
- **SC-006**: 配置页面的加载时间不超过2秒
- **SC-007**: 配置保存的响应时间不超过1秒

## Assumptions

- 管理员具有基本的推荐系统知识，了解任务类型和奖励类型的含义
- 系统已经存在推荐系统的基础数据表和模型
- 系统已经存在配置保存的后端接口
- 管理员使用现代浏览器（Chrome、Firefox、Safari、Edge）访问配置页面
- 配置页面使用 Amaze UI 框架进行样式设计
