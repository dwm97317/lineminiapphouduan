# LINE 消息模板变量编辑器使用指南

## 功能概述

商户后台现在支持可视化编辑消息模板变量，可以从预定义的变量列表中选择需要在消息中显示的字段。

## 功能特点

### ✅ 可视化变量选择
- 每个消息类型都有专属的可用变量列表
- 复选框界面，直观易用
- 实时预览已选变量

### ✅ 变量分类
- **必需变量**（标记 *）：核心字段，无法取消，自动包含
- **可选变量**：根据需求自由选择

### ✅ 详细说明
每个变量都包含：
- **中文标签**：变量的中文名称
- **变量名**：在模板中使用的代码（如 `{{shop_name}}`）
- **示例值**：帮助理解变量内容

## 使用方法

### 1. 选择变量

1. 进入 **商户后台 > 设置 > LINE 配置**
2. 切换到 **消息通知 (Messaging API)** 标签
3. 找到要配置的消息模板（如"包裹入库通知"）
4. 在"模板变量"区域，勾选需要的变量
5. 必需变量（标记 *）已默认选中且无法取消

### 2. 实时预览

- 勾选或取消变量后，下方的"当前已选变量"会实时更新
- 显示所有已选变量的列表

### 3. 保存配置

1. 配置完所有字段后，点击底部的"提交保存"按钮
2. 系统会自动保存选中的变量到数据库
3. 保存成功后，消息发送时会使用新的变量配置

### 4. 在模板中使用变量

在 Flex Message 模板中使用 `{{变量名}}` 格式引用变量：

```json
{
  "type": "text",
  "text": "仓库：{{shop_name}}"
}
```

系统会自动将 `{{shop_name}}` 替换为实际的仓库名称。

## 技术实现

### 前端
- 复选框界面，支持多选
- JavaScript 实时更新预览
- 必需变量自动禁用（disabled + checked）

### 后端
- 控制器定义所有可用变量及其元数据
- 保存时将选中的变量转换为数组存储
- 发送消息时根据选中的变量渲染模板

### 数据存储
```php
'templates' => [
    'inwarehouse' => [
        'variables' => ['shop_name', 'express_num', 'weight', ...],
        // 其他配置...
    ]
]
```

## 优势

1. **灵活性**：商户可以根据需求自定义显示的信息
2. **简洁性**：只显示必要的信息，避免消息过长
3. **可扩展**：后续可以轻松添加新变量
4. **用户友好**：可视化界面，无需编写代码

## 注意事项

1. **必需变量**：标记 * 的变量是消息的核心信息，无法取消
2. **模板兼容**：修改变量后，需确保 Flex Message 模板中使用了对应的变量
3. **数据可用性**：确保后端在发送消息时能提供所有选中变量的数据
