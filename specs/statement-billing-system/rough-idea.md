# 集运订单账单归档系统 - 初始想法

## 背景

当前集运订单系统（yoshop_package）包含完整的物流和费用信息，但缺少对已结算订单的归档管理。需要建立一套账单系统，实现：

1. 财务对账的周期性管理
2. 已出账订单与待出账订单的数据隔离
3. 历史账单的可追溯性
4. 灵活的计价配置（支持多种计价方式）

## 核心需求

### 1. 财务配置管理
- 账单模板配置（LOGO、二维码、提示文字）
- 全局默认单价配置
- 客户单价配置（5种计价方式：固定单价、阶梯价格、按线路、按重量区间、自定义公式）
- 历史单价表导入（支持TXT/Excel批量导入）

### 2. 账单生成功能
- 查询待出账订单（statement_id IS NULL）
- 批量选择订单生成账单
- 自动计算金额（根据配置的计价方式）
- 生成Excel账单文件
- 支持账单编号自动生成（ST+日期+流水号）

### 3. 账单管理功能
- 查看账单详情
- 标记账单为已支付（同步更新订单支付状态）
- 作废账单（解除订单绑定）
- 下载账单Excel

### 4. Excel生成模块
- 美观的Excel模板（包含LOGO、二维码、订单明细、汇总）
- 支持大批量订单（性能优化）
- 图片插入功能

## 技术约束

### 现有系统
- **后端**：PHP (ThinkPHP 5.0)
- **前端**：Amaze UI + jQuery
- **数据库**：MySQL 5.7+
- **现有问题**：Package/Index.php 已有1457行代码，不能继续添加

### 架构要求
- **不修改现有代码**：保持 Package/Index.php 稳定
- **服务层模式**：业务逻辑放服务层，控制器保持薄
- **单一职责**：每个类只做一件事
- **可测试性**：服务层可独立测试

## 成功标准

1. 财务可以灵活配置客户单价
2. 账单生成准确无误
3. 账单支付状态与订单状态保持一致
4. Excel文件格式美观、数据完整
5. 代码结构清晰、易于维护
6. 不影响现有集运订单功能
