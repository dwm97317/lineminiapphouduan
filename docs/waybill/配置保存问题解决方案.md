# 电子面单配置保存问题 - 解决方案

## 问题

用户反馈保存 API 配置时收到错误：
```json
{code: 0, msg: "保存失败", url: "", data: []}
```

## 根本原因

1. **静态方法冲突**：`app\common\model\Setting` 添加的静态 `edit()` 方法与 `app\store\model\Setting` 的实例方法冲突
2. **保存逻辑不完善**：直接使用基础模型保存，缺少必要的数据验证和处理

## 已实施的修复

### 修复 1: 删除冲突的静态方法

从 `source/application/common/model/Setting.php` 中删除了静态 `edit()` 方法。

### 修复 2: 更新 WaybillConfigService

修改 `source/application/common/service/WaybillConfigService.php` 的 `saveConfig()` 方法，使用实例方法而非静态方法。

### 修复 3: 改进 saveApiConfig 方法 ✅ **推荐方案**

修改 `source/application/store/controller/setting/WaybillConfig.php`：

```php
use app\store\model\Setting as StoreSettingModel;

public function saveApiConfig()
{
    $config = $this->request->post('config');
    
    if (empty($config)) {
        return $this->renderError('参数错误：配置数据为空');
    }

    if (is_string($config)) {
        $config = json_decode($config, true);
    }
    
    if (!$config || !is_array($config)) {
        return $this->renderError('配置数据格式错误');
    }

    try {
        $model = new StoreSettingModel();
        $result = $model->edit('waybill', $config);
        
        if ($result) {
            return $this->renderSuccess([], '保存成功');
        }
        
        $error = $model->getError();
        return $this->renderError($error ?: '保存失败');
        
    } catch (\Exception $e) {
        return $this->renderError('保存失败：' . $e->getMessage());
    }
}
```

## 为什么使用 StoreSettingModel::edit()

`app\store\model\Setting::edit()` 方法包含了完善的业务逻辑：

1. ✅ **自动数据验证**：`validValues()` 方法
2. ✅ **自动处理特殊字段**：weight_mode、price_mode、size_mode 等
3. ✅ **自动缓存清理**：保存后清除相关缓存
4. ✅ **wxapp_id 管理**：自动使用当前店铺的 wxapp_id
5. ✅ **错误处理**：统一的错误信息返回
