# 电子面单配置保存问题排查指南

## 问题描述

用户在保存 API 配置时收到错误：`{code: 0, msg: "保存失败", url: "", data: []}`

## 已完成的修复

### 1. 修复了静态方法冲突
- 删除了 `app\common\model\Setting::edit()` 静态方法
- 更新了 `WaybillConfigService` 使用实例方法

### 2. 增强了 saveApiConfig 方法
- 添加了详细的错误信息
- 改进了 wxapp_id 获取逻辑
- 添加了异常捕获

### 3. 当前 saveApiConfig 逻辑

```php
public function saveApiConfig()
{
    // 1. 获取并验证配置数据
    $config = $this->request->post('config');
    
    // 2. JSON 解码（如果是字符串）
    if (is_string($config)) {
        $config = json_decode($config, true);
    }
    
    // 3. 获取 wxapp_id
    $wxapp_id = isset($this->store['wxapp_id']) 
        ? $this->store['wxapp_id'] 
        : SettingModel::$wxapp_id;
    
    // 4. 查找或创建记录
    $model = SettingModel::get(['key' => 'waybill', 'wxapp_id' => $wxapp_id]);
    
    if ($model) {
        // 更新现有记录
        $result = $model->save([
            'values' => $config,
            'describe' => '电子面单配置',
            'update_time' => time()
        ]);
    } else {
        // 创建新记录
        $model = new SettingModel();
        $result = $model->save([
            'key' => 'waybill',
            'values' => $config,
            'describe' => '电子面单配置',
            'wxapp_id' => $wxapp_id,
            'update_time' => time()
        ]);
    }
    
    // 5. 清除缓存并返回
    if ($result !== false) {
        \think\Cache::rm('setting_' . $wxapp_id);
        return $this->renderSuccess([], '保存成功');
    }
}
```

## 可能的原因

### 原因 1: wxapp_id 未正确获取

**症状：** 错误信息显示"无法获取 wxapp_id"

**解决方案：**
```php
// 在 WaybillConfig 控制器的 _initialize 方法中
public function _initialize()
{
    parent::_initialize();
    
    // 确保 wxapp_id 存在
    if (!isset($this->store['wxapp_id'])) {
        $this->store['wxapp_id'] = 10001; // 使用默认值
    }
    
    $this->configService = new WaybillConfigService($this->store['wxapp_id']);
}
```

### 原因 2: 数据库表字段问题

**症状：** SQL 错误

**检查：**
```sql
DESCRIBE yoshop_setting;
```

确保有以下字段：
- `key` (varchar)
- `values` (text)
- `describe` (varchar)
- `wxapp_id` (int)
- `update_time` (int)

### 原因 3: JSON 序列化问题

**症状：** values 字段保存为 null

**检查 Setting 模型：**
```php
// 应该有这两个方法
public function getValuesAttr($value)
{
    return json_decode($value, true);
}

public function setValuesAttr($value)
{
    return json_encode($value);
}
```

### 原因 4: 缓存问题

**解决方案：**
```bash
# 清空缓存目录
rm -rf runtime/cache/*

# Windows
rd /s /q runtime\cache
mkdir runtime\cache
```

## 快速修复方案

如果上述排查仍无法解决，可以使用以下临时方案：

### 方案 A: 直接操作数据库

修改 `saveApiConfig` 方法：

```php
public function saveApiConfig()
{
    $config = $this->request->post('config');
    
    if (is_string($config)) {
        $config = json_decode($config, true);
    }
    
    if (!$config) {
        return $this->renderError('配置数据格式错误');
    }

    // 直接使用 DB 操作
    $wxapp_id = isset($this->store['wxapp_id']) ? $this->store['wxapp_id'] : 10001;
    
    $result = \think\Db::name('setting')
        ->where(['key' => 'waybill', 'wxapp_id' => $wxapp_id])
        ->update([
            'values' => json_encode($config),
            'update_time' => time()
        ]);
    
    if ($result !== false) {
        \think\Cache::rm('setting_' . $wxapp_id);
        return $this->renderSuccess([], '保存成功');
    }
    
    return $this->renderError('保存失败');
}
```

### 方案 B: 使用 store 模型的 edit 方法

修改 `saveApiConfig` 方法使用 `app\store\model\Setting`：

```php
use app\store\model\Setting as StoreSettingModel;

public function saveApiConfig()
{
    $config = $this->request->post('config');
    
    if (is_string($config)) {
        $config = json_decode($config, true);
    }
    
    if (!$config) {
        return $this->renderError('配置数据格式错误');
    }

    $model = new StoreSettingModel();
    $result = $model->edit('waybill', $config);
    
    if ($result) {
        return $this->renderSuccess([], '保存成功');
    }
    
    return $this->renderError('保存失败');
}
```
