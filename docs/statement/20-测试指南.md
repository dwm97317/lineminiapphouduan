# 财务数据导入功能测试指南

## 概述

本文档说明如何测试财务数据导入功能的各个组件。

## 测试文件

### 1. test_payment_import_generator.php
**用途**: 生成测试用的Excel文件

**运行方法**:
```bash
php test_payment_import_generator.php
```

**生成的文件**:
- `test_basic_colors.xlsx` - 基础颜色测试（蓝色、粉红色、绿色、白色、未知颜色）
- `test_member_id_formats.xlsx` - Member_ID格式测试（带国家后缀、纯数字、带空格）
- `test_matching_scenarios.xlsx` - 匹配场景测试（精确匹配、模糊匹配、多重匹配、无匹配）
- `test_multi_sheet.xlsx` - 多Sheet测试
- `test_edge_cases.xlsx` - 边界情况测试
- `test_date_formats.xlsx` - 日期格式测试

### 2. test_white_row_filtering.php
**用途**: 验证白色行过滤功能

**运行方法**:
```bash
php test_white_row_filtering.php
```

**测试内容**:
- 白色行检测（FFFFFF）
- 无填充行检测（000000）
- 白色行跳过逻辑
- 非白色行正常处理

### 3. test_comprehensive_import.php
**用途**: 综合测试所有导入功能

**运行方法**:
```bash
php test_comprehensive_import.php
```

**测试内容**:
1. Excel解析功能
   - 文件加载
   - 表头验证
   - 数据读取

2. 颜色检测功能
   - 蓝色检测（已支付+已出账）
   - 粉红色检测（已出账未支付）
   - 绿色检测（已支付未出账）
   - 白色检测（未处理）
   - 未知颜色检测

3. Member_ID解析功能
   - 纯数字格式
   - 带国家后缀格式
   - 带空格格式

4. 日期解析功能
   - 中文日期格式（X月X日）
   - 空日期处理

5. 白色行过滤功能
   - 白色行跳过
   - 非白色行处理

6. 订单匹配功能
   - 精确匹配（通过国际单号）
   - 模糊匹配（通过Member_ID+重量）
   - 重量容差匹配（±0.5kg）
   - 无匹配场景

### 失败处理

如果测试失败，检查：
1. 测试文件是否已生成（运行 test_payment_import_generator.php）
2. 颜色检测逻辑是否正确
3. Member_ID解析是否正确
4. 订单匹配逻辑是否正确

## 颜色检测规则

### 蓝色（已支付+已出账）
- B > 200
- B > R + 10
- B > G + 10

### 粉红色（已出账未支付）
- R > 200
- G > 150
- B > 150
- R > G 且 R > B

### 绿色（已支付未出账）
- G > 200
- G > R + 30
- G > B + 30

### 白色（未处理）
- R=255, G=255, B=255（纯白色）
- R=0, G=0, B=0（无填充）

## 订单匹配规则

### 精确匹配
- 有国际单号：通过 `express_num` 精确匹配

### 模糊匹配
- 无国际单号：通过 `user_id` + `weight` 匹配
- 重量容差：±0.5kg

### 匹配优先级
1. 国际单号精确匹配
2. Member_ID + 重量模糊匹配

## 常见问题
