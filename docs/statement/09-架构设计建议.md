# 账单系统架构设计建议

## 一、现状分析

### 1.1 当前问题
- `Package/Index.php` 已有 **1457行代码**
- 单个文件承担过多职责（上帝类）
- 新增功能会进一步增加复杂度
- 维护困难，测试困难

### 1.2 风险
- 代码耦合度高
- 修改一处可能影响其他功能
- 团队协作冲突多
- 难以进行单元测试

### 2.2 职责划分

#### 控制器层（Controller）
**职责**：接收HTTP请求，调用服务层，返回响应
**原则**：薄控制器，不包含业务逻辑

```php
// 示例：Package/Statement.php
class Statement extends Controller
{
    /**
     * 账单详情
     */
    public function detail()
    {
        $statementId = $this->request->param('statement_id');
        
        try {
            $service = new StatementManageService();
            $data = $service->getStatementDetail($statementId);
            
            return $this->renderSuccess('', $data);
        } catch (\Exception $e) {
            return $this->renderError($e->getMessage());
        }
    }
}
```

#### 服务层（Service）
**职责**：业务逻辑处理
**原则**：单一职责，可复用，可测试

```php
// 示例：StatementManageService.php
class StatementManageService
{
    /**
     * 获取账单详情
     */
    public function getStatementDetail($statementId)
    {
        // 业务逻辑
        $statement = Statement::find($statementId);
        $packages = Package::where('statement_id', $statementId)->select();
        
        return [
            'statement' => $statement,
            'packages' => $packages
        ];
    }
}
```

#### 模型层（Model）
**职责**：数据访问和基本数据操作
**原则**：只包含数据相关的逻辑

```php
// 示例：Statement.php
class Statement extends Model
{
    /**
     * 标记为已支付
     */
    public function markAsPaid($remark = '')
    {
        Db::startTrans();
        try {
            $this->save([
                'pay_status' => 2,
                'pay_time' => date('Y-m-d H:i:s'),
                'pay_remark' => $remark
            ]);
            
            Package::where('statement_id', $this->id)
                ->update(['is_pay' => 1]);
            
            Db::commit();
            return true;
        } catch (\Exception $e) {
            Db::rollback();
            throw $e;
        }
    }
}
```

## 三、实施方案

### 3.1 阶段1：创建新模块（不影响现有功能）

**步骤**：
1. 创建新的控制器 `Package/Statement.php`
2. 创建新的控制器 `Finance/Config.php`
3. 创建服务层类
4. 创建模型类
5. 创建视图文件

**优点**：
- 不修改现有的 `Package/Index.php`
- 新旧功能隔离
- 风险低

### 3.2 阶段2：集成到现有页面

**步骤**：
1. 在 `Package/Index.php` 视图中添加账单相关按钮
2. 通过Ajax调用新的 `Statement` 控制器
3. 使用弹窗显示账单详情

**示例**：
```html
<!-- 在 package/index.php 中 -->
<button class="am-btn am-btn-primary" onclick="viewStatement(<?= $item['statement_id'] ?>)">
    查看账单
</button>

<script>
function viewStatement(statementId) {
    $.ajax({
        url: '/store/package.statement/detail',
        data: { statement_id: statementId },
        success: function(res) {
            if (res.code == 1) {
                // 显示弹窗
                showStatementModal(res.data);
            }
        }
    });
}
</script>
```

### 3.3 阶段3：重构现有代码（可选）

如果时间允许，可以逐步重构 `Package/Index.php`：
1. 提取业务逻辑到服务层
2. 控制器只保留请求处理
3. 提高代码可测试性

## 五、代码示例

### 5.1 控制器示例（薄控制器）

```php
<?php
namespace app\store\controller\package;

use app\store\controller\Controller;
use app\store\service\statement\StatementManageService;

class Statement extends Controller
{
    private $service;
    
    public function __construct()
    {
        parent::__construct();
        $this->service = new StatementManageService();
    }
    
    /**
     * 账单列表
     */
    public function index()
    {
        $list = $this->service->getStatementList($this->request->param());
        return $this->renderSuccess('', compact('list'));
    }
    
    /**
     * 账单详情
     */
    public function detail()
    {
        $statementId = $this->request->param('statement_id');
        
        try {
            $data = $this->service->getStatementDetail($statementId);
            return $this->renderSuccess('', $data);
        } catch (\Exception $e) {
            return $this->renderError($e->getMessage());
        }
    }
    
    /**
     * 标记为已支付
     */
    public function markAsPaid()
    {
        $statementId = $this->request->post('statement_id');
        $remark = $this->request->post('remark', '');
        
        try {
            $this->service->markStatementAsPaid($statementId, $remark);
            return $this->renderSuccess('标记成功');
        } catch (\Exception $e) {
            return $this->renderError($e->getMessage());
        }
    }
}
```

### 5.2 服务层示例（业务逻辑）

```php
<?php
namespace app\store\service\statement;

use app\store\model\Statement;
use app\store\model\Package;

class StatementManageService
{
    private $wxapp_id;
    
    public function __construct()
    {
        $this->wxapp_id = helper::getWxappId();
    }
    
    /**
     * 获取账单列表
     */
    public function getStatementList($params = [])
    {
        $model = Statement::where('wxapp_id', $this->wxapp_id);
        
        // 筛选条件
        if (!empty($params['member_id'])) {
            $model->where('member_id', $params['member_id']);
        }
        
        if (isset($params['pay_status']) && $params['pay_status'] !== '') {
            $model->where('pay_status', $params['pay_status']);
        }
        
        return $model->order('id', 'desc')->paginate(15);
    }
    
    /**
     * 获取账单详情
     */
    public function getStatementDetail($statementId)
    {
        $statement = Statement::where('id', $statementId)
            ->where('wxapp_id', $this->wxapp_id)
            ->find();
        
        if (!$statement) {
            throw new \Exception('账单不存在');
        }
        
        $packages = Package::where('statement_id', $statementId)
            ->where('is_delete', 0)
            ->select();
        
        return [
            'statement' => $statement,
            'packages' => $packages,
            'statistics' => $statement->getStatistics()
        ];
    }
    
    /**
     * 标记账单为已支付
     */
    public function markStatementAsPaid($statementId, $remark = '')
    {
        $statement = Statement::where('id', $statementId)
            ->where('wxapp_id', $this->wxapp_id)
            ->find();
        
        if (!$statement) {
            throw new \Exception('账单不存在');
        }
        
        return $statement->markAsPaid($remark);
    }
}
```

## 六、优势总结

### 6.1 可维护性
- ✅ 代码职责清晰
- ✅ 修改影响范围小
- ✅ 易于定位问题

### 6.2 可测试性
- ✅ 服务层可独立测试
- ✅ 不依赖HTTP请求
- ✅ 易于Mock依赖

### 6.3 可扩展性
- ✅ 新增功能不影响现有代码
- ✅ 服务层可复用
- ✅ 易于添加新的计价方式

### 6.4 团队协作
- ✅ 不同开发者负责不同模块
- ✅ 减少代码冲突
- ✅ 代码审查更容易

## 七、迁移建议

### 7.1 不要一次性重构
- ❌ 不要试图一次性重构整个 `Package/Index.php`
- ✅ 新功能使用新架构
- ✅ 旧功能保持稳定
- ✅ 逐步迁移

### 7.2 保持向后兼容
- ✅ 不修改现有API
- ✅ 不影响现有功能
- ✅ 新旧代码可以共存

### 7.3 充分测试
- ✅ 新功能完整测试
- ✅ 回归测试现有功能
- ✅ 集成测试
