## 测试场景

### 场景1: 正常流程

**Given**: 存在3个待出账订单，属于同一客户  
**When**: 调用 `createStatement([1,2,3], 31398)`  
**Then**: 
- 生成账单记录
- 账单编号格式正确（ST20260228001）
- 订单被标记为已归档（statement_id不为空）
- 金额计算正确

**测试代码**:
```php
$service = new StatementService();
$statement = $service->createStatement([1, 2, 3], 31398);

// 验证
assert($statement['statement_no'] !== null);
assert($statement['total_packages'] == 3);
assert($statement['total_amount'] > 0);

// 验证订单绑定
$package = Package::find(1);
assert($package['statement_id'] == $statement['id']);
```

---

### 场景2: 并发控制

**Given**: 订单已被归档到账单A  
**When**: 尝试再次为相同订单生成账单B  
**Then**: 抛出异常"订单已归档"

**测试代码**:
```php
try {
    $service->createStatement([1, 2, 3], 31398);
    assert(false, '应该抛出异常');
} catch (\Exception $e) {
    assert(strpos($e->getMessage(), '已归档') !== false);
}
```

---

### 场景3: 不同客户订单

**Given**: 订单1属于客户A，订单2属于客户B  
**When**: 尝试为订单[1,2]生成账单  
**Then**: 抛出异常"只能选择同一个客户的订单"

**测试代码**:
```php
try {
    $service->createStatement([1, 100], 31398);
    assert(false, '应该抛出异常');
} catch (\Exception $e) {
    assert(strpos($e->getMessage(), '同一个客户') !== false);
}
```

---

### 场景4: 计价方式测试

**Given**: 客户配置了阶梯价格（0-10KG: 50元，10+KG: 46元）  
**When**: 生成包含8KG和15KG订单的账单  
**Then**: 
- 8KG订单使用50元/KG
- 15KG订单使用46元/KG

**测试代码**:
```php
// 先配置阶梯价格
FinanceConfig::create([
    'member_id' => 31398,
    'price_type' => 2,
    'price_tier_json' => json_encode([
        'tiers' => [
            ['min' => 0, 'max' => 10, 'price' => 50],
            ['min' => 10, 'max' => null, 'price' => 46]
        ]
    ]),
    'status' => 1,
    'wxapp_id' => 10001
]);

// 生成账单
$statement = $service->createStatement([1, 2], 31398);

// 验证金额
// 8KG * 50 + 15KG * 46 = 400 + 690 = 1090
assert($statement['total_amount'] == 1090);
```

### 场景5: 账单编号唯一性

**Given**: 系统正常运行  
**When**: 连续生成10个账单  
**Then**: 所有账单编号唯一

**测试代码**:
```php
$numbers = [];
for ($i = 0; $i < 10; $i++) {
    $no = Statement::generateStatementNo();
    $numbers[] = $no;
}

$unique = array_unique($numbers);
assert(count($unique) == 10);
```

## 常见问题

### Q1: 提示"订单不可用或已归档"

**原因**: 订单已被其他账单归档，或订单不存在  
**解决**: 查询 `statement_id IS NULL` 的订单

### Q2: 提示"只能选择同一个客户的订单"

**原因**: 选择的订单属于不同客户  
**解决**: 确保所有订单的 `member_id` 相同

### Q3: 金额计算不正确

**原因**: 计价配置错误或订单重量为0  
**解决**: 检查 `yoshop_finance_config` 表和订单重量
