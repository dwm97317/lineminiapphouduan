# 历史单价表导入功能说明

## 一、功能背景

财务以前使用TXT或Excel文件维护所有客户的单价配置，文件中包含了几百个客户的ID和对应单价。例如：

**TXT文件示例（price_list.txt）**：
```
31398 46.00
31399 48.00
31400 50.00
31401 52.00
31402 46.00
31403 48.50
... (可能有几百行)
```

**Excel文件示例（price_list.xlsx）**：
```
A列      B列
31398   46.00
31399   48.00
31400   50.00
31401   52.00
31402   46.00
31403   48.50
... (可能有几百行)
```

## 二、导入功能设计

### 2.1 文件格式支持

**TXT格式**：
- 每行一条记录
- 格式：`客户ID 单价`
- 分隔符：空格或制表符（Tab）
- 示例：`31398 46.00`
- 可以包含几百甚至上千条记录

**Excel格式**：
- A列：客户ID
- B列：单价（元/KG）
- 可以有表头也可以没有（程序会自动识别数字）
- 支持.xls和.xlsx格式
- 可以包含几百甚至上千条记录

### 2.2 导入流程

```
1. 财务准备历史单价文件（TXT或Excel）
   ↓
2. 在系统中点击"导入历史单价表"
   ↓
3. 选择文件并上传
   ↓
4. 系统自动解析文件内容
   ↓
5. 将所有客户ID和单价存储为JSON格式
   ↓
6. 导入成功，显示导入的客户数量
```

### 2.3 数据存储

导入后，所有客户单价数据会以JSON格式存储在数据库中：

```json
{
  "31398": 46.00,
  "31399": 48.00,
  "31400": 50.00,
  "31401": 52.00,
  "31402": 46.00,
  "31403": 48.50,
  ...
}
```

这样可以快速查询任意客户的单价。

## 三、使用场景

### 场景1：系统初始化
```
财务有一个包含200个客户单价的Excel文件
→ 直接导入到系统
→ 所有200个客户的单价立即生效
→ 无需手动逐个录入
```

### 场景2：批量更新单价
```
财务更新了单价文件（修改了部分客户单价）
→ 重新导入文件
→ 系统覆盖旧数据
→ 新单价立即生效
```

### 场景3：与专属配置结合
```
历史单价表：包含200个客户的单价
客户专属配置：为5个VIP客户单独配置阶梯价格
→ 这5个VIP使用专属配置（优先级更高）
→ 其余195个客户使用历史单价表
→ 新客户使用全局默认单价
```

## 四、配置优先级

```
生成账单时查询客户单价：

1. 查询客户专属配置
   ↓ 如果存在，使用专属配置（如阶梯价格）
   ↓ 如果不存在
   
2. 查询历史单价表
   ↓ 如果客户ID在历史表中，使用历史表单价
   ↓ 如果不在历史表中
   
3. 使用全局默认单价
   ↓ 如果全局默认已配置，使用全局默认
   ↓ 如果未配置
   
4. 使用系统兜底单价（46元/KG）
```

## 六、技术实现要点

### 6.1 文件解析

**TXT文件解析**：
```php
$lines = file($filePath, FILE_IGNORE_NEW_LINES | FILE_SKIP_EMPTY_LINES);
foreach ($lines as $line) {
    $parts = preg_split('/\s+/', trim($line), 2);
    if (count($parts) == 2) {
        $memberId = intval($parts[0]);
        $unitPrice = floatval($parts[1]);
        if ($memberId > 0 && $unitPrice > 0) {
            $priceData[$memberId] = $unitPrice;
        }
    }
}
```

**Excel文件解析**：
```php
$spreadsheet = \PhpOffice\PhpSpreadsheet\IOFactory::load($filePath);
$worksheet = $spreadsheet->getActiveSheet();
$highestRow = $worksheet->getHighestRow();

for ($row = 1; $row <= $highestRow; $row++) {
    $memberId = intval($worksheet->getCell('A' . $row)->getValue());
    $unitPrice = floatval($worksheet->getCell('B' . $row)->getValue());
    if ($memberId > 0 && $unitPrice > 0) {
        $priceData[$memberId] = $unitPrice;
    }
}
```

### 6.2 数据查询

```php
// 从历史单价表获取客户单价
public static function getPriceFromHistory($memberId)
{
    $historyTable = self::getHistoryPriceTable();
    
    if (!$historyTable || empty($historyTable['price_data_json'])) {
        return null;
    }
    
    $priceData = json_decode($historyTable['price_data_json'], true);
    
    // 直接通过客户ID查询
    if (isset($priceData[$memberId])) {
        return [
            'price_type' => self::PRICE_TYPE_FIXED,
            'unit_price' => $priceData[$memberId]
        ];
    }
    
    return null;
}
```

### 6.3 性能优化

- JSON格式存储，查询速度快（O(1)复杂度）
- 一次导入，多次使用
- 支持几千条记录也不影响性能

## 七、常见问题

### Q1: 文件中有重复的客户ID怎么办？
A: 系统会自动去重，保留最后一次出现的单价。

### Q2: 文件格式不对会怎样？
A: 系统会提示"文件内容为空或格式错误"，不会导入。

### Q3: 导入后可以修改吗？
A: 可以重新导入文件覆盖，或者为特定客户配置专属单价。

### Q4: 历史单价表会影响已生成的账单吗？
A: 不会，已生成的账单数据已固化，不受配置变更影响。

### Q5: 可以导出当前的历史单价表吗？
A: 可以，点击"查看明细"后可以导出为Excel文件。

## 八、使用建议

1. **首次使用**：如果有历史单价文件，先导入历史单价表
2. **定期更新**：单价调整后，更新文件并重新导入
3. **特殊客户**：为VIP客户单独配置专属计价方式
4. **新客户**：新客户会自动使用全局默认单价
5. **备份文件**：保留历史单价文件的备份，便于恢复

## 十、总结

历史单价表导入功能是为了方便财务快速迁移现有的单价配置，避免手动逐个录入。一次导入，所有客户的单价立即生效，大大提高了工作效率。
