## 一、数据表设计

### 1.1 账单主表（yoshop_statement）

```sql
CREATE TABLE yoshop_statement (
  id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '账单ID',
  statement_no VARCHAR(50) NOT NULL UNIQUE COMMENT '账单编号（如：ST202601280001）',
  member_id INT NOT NULL COMMENT '客户ID',
  member_name VARCHAR(100) COMMENT '客户姓名',
  start_date DATE NOT NULL COMMENT '账单起始日期',
  end_date DATE NOT NULL COMMENT '账单截止日期',
  total_packages INT DEFAULT 0 COMMENT '包裹总数',
  total_weight DECIMAL(12,3) DEFAULT 0 COMMENT '总计费重量',
  total_amount DECIMAL(10,2) DEFAULT 0 COMMENT '总金额',
  status TINYINT DEFAULT 1 COMMENT '1=草稿 2=已确认 3=已发送',
  pay_status TINYINT DEFAULT 1 COMMENT '1=未支付 2=已支付',
  pay_time DATETIME COMMENT '支付时间',
  pay_remark VARCHAR(255) COMMENT '支付备注',
  excel_path VARCHAR(255) COMMENT 'Excel文件路径',
  created_by INT COMMENT '创建人ID',
  created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  confirmed_time DATETIME COMMENT '确认时间',
  wxapp_id INT NOT NULL COMMENT '所属商家',
  INDEX idx_member_date (member_id, start_date, end_date),
  INDEX idx_wxapp (wxapp_id),
  INDEX idx_pay_status (pay_status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='集运订单账单表';
```

**字段说明**：
- `statement_no`：账单编号，格式：ST + 年月日 + 4位流水号
- `status`：账单状态，用于流程控制
- `pay_status`：支付状态，核心字段，驱动订单支付状态更新
- `total_*`：汇总字段，冗余存储便于查询

### 1.2 财务配置表（yoshop_finance_config）

```sql
CREATE TABLE yoshop_finance_config (
  id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '配置ID',
  config_type TINYINT NOT NULL COMMENT '配置类型：1=账单模板 2=客户单价 3=全局默认单价',
  config_name VARCHAR(100) NOT NULL COMMENT '配置名称',
  
  -- 账单模板配置字段
  logo_path VARCHAR(255) COMMENT 'LOGO图片路径',
  title VARCHAR(100) COMMENT '账单标题（如：泰国-中国）',
  alipay_qr_path VARCHAR(255) COMMENT '支付宝收款码路径',
  wechat_qr_path VARCHAR(255) COMMENT '微信收款码路径',
  alipay_qr2_path VARCHAR(255) COMMENT '支付宝收款码2路径',
  notice_text TEXT COMMENT '账单提示文字',
  
  -- 客户单价配置字段（客户专属或全局默认）
  member_id INT DEFAULT NULL COMMENT '客户ID（NULL表示全局默认）',
  price_type TINYINT DEFAULT 1 COMMENT '计价方式：1=固定单价 2=阶梯价格 3=按线路 4=按重量区间 5=自定义公式',
  unit_price DECIMAL(10,2) COMMENT '固定单价（元/KG）',
  price_tier_json TEXT COMMENT '阶梯价格JSON',
  price_line_json TEXT COMMENT '线路价格JSON',
  price_range_json TEXT COMMENT '重量区间价格JSON',
  price_formula VARCHAR(500) COMMENT '自定义计价公式',
  
  -- 通用字段
  is_default TINYINT DEFAULT 0 COMMENT '是否默认配置',
  sort INT DEFAULT 0 COMMENT '排序',
  status TINYINT DEFAULT 1 COMMENT '状态：1=启用 0=禁用',
  remark VARCHAR(255) COMMENT '备注',
  created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  wxapp_id INT NOT NULL COMMENT '所属商家',
  
  INDEX idx_type (config_type),
  INDEX idx_member (member_id),
  INDEX idx_wxapp (wxapp_id),
  UNIQUE KEY uk_member_price (member_id, wxapp_id, config_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='财务配置表';
```

**字段说明**：
- `config_type`：配置类型，一表多用
- `member_id`：NULL表示全局配置，有值表示客户专属配置
- `price_*_json`：JSON格式存储复杂配置
- `uk_member_price`：唯一索引，确保每个客户只有一个单价配置

### 1.3 修改集运订单表（yoshop_package）

```sql
ALTER TABLE yoshop_package 
ADD COLUMN statement_id INT DEFAULT NULL COMMENT '所属账单ID',
ADD INDEX idx_statement (statement_id);
```

**说明**：
- `statement_id`：关联账单ID，NULL表示未出账
- 索引用于快速查询账单下的所有订单

## 四、索引优化建议

### 4.1 查询场景分析

**场景1：查询待出账订单**
```sql
SELECT * FROM yoshop_package 
WHERE member_id = ? 
  AND is_delete = 0 
  AND statement_id IS NULL
  AND entering_warehouse_time BETWEEN ? AND ?;
```
**索引**：已有 `idx_member_status_delete_time`

**场景2：查询账单下的所有订单**
```sql
SELECT * FROM yoshop_package WHERE statement_id = ?;
```
**索引**：新增 `idx_statement`

**场景3：查询客户的账单列表**
```sql
SELECT * FROM yoshop_statement 
WHERE member_id = ? 
  AND start_date >= ? 
  AND end_date <= ?;
```

## 五、数据一致性检查

### 5.1 检查脚本

```sql
-- 检查账单已支付但订单未支付的异常数据
SELECT p.id, p.express_num, s.statement_no, s.pay_status, p.is_pay
FROM yoshop_package p
JOIN yoshop_statement s ON p.statement_id = s.id
WHERE s.pay_status = 2 AND p.is_pay = 0;

-- 检查账单汇总数据是否正确
SELECT 
  s.id,
  s.statement_no,
  s.total_packages,
  COUNT(p.id) AS actual_packages,
  s.total_amount,
  SUM(p.real_payment) AS actual_amount
FROM yoshop_statement s
LEFT JOIN yoshop_package p ON s.id = p.statement_id
GROUP BY s.id
HAVING s.total_packages != actual_packages 
    OR ABS(s.total_amount - actual_amount) > 0.01;
```

### 5.2 修复脚本

```sql
-- 修复账单汇总数据
UPDATE yoshop_statement s
SET 
  total_packages = (SELECT COUNT(*) FROM yoshop_package WHERE statement_id = s.id),
  total_weight = (SELECT SUM(weight) FROM yoshop_package WHERE statement_id = s.id),
  total_amount = (SELECT SUM(real_payment) FROM yoshop_package WHERE statement_id = s.id)
WHERE s.id = ?;
```
